# GUÍA DE DESPLIEGUE LOCAL - SUPERMERCADO CIMA (ANDROID)

Esta guía explica cómo configurar la aplicación para que funcione en una red local (Wi-Fi de la empresa/tienda), de modo que las tablets se conecten a un ordenador central que servirá como base de datos.

---

## 1. PREPARAR EL ORDENADOR CENTRAL (SERVIDOR)

Este ordenador debe estar encendido siempre que se quiera usar la aplicación.

### A. Instalar Node.js
1. Descarga e instala la versión "LTS" de Node.js desde: https://nodejs.org/
2. Verifica la instalación abriendo una terminal y escribiendo: `node -v`

### B. Fijar la Dirección IP (MUY IMPORTANTE)
Para evitar que la aplicación deje de funcionar si el router se reinicia, debes asignar una IP fija al ordenador:
1. Ve a **Panel de Control** > **Centro de redes y recursos compartidos**.
2. Haz clic en tu conexión (Wi-Fi o Ethernet) > **Propiedades**.
3. Selecciona "Protocolo de Internet versión 4 (TCP/IPv4)" y pulsa **Propiedades**.
4. Selecciona "Usar la siguiente dirección IP".
5. Introduce una IP (ej: `192.168.1.100`), la Máscara de subred (`255.255.255.0`) y la Puerta de enlace (suele ser `192.168.1.1`).
6. Pulsa Aceptar. *Anota la IP que has puesto.*

### C. Configurar el Firewall de Windows
Para que las tablets puedan "ver" al ordenador:
1. Busca "Firewall de Windows con seguridad avanzada" en el inicio.
2. Haz clic en **Reglas de entrada** > **Nueva regla**.
3. Selecciona **Puerto** > Siguiente.
4. Elige **TCP** y en "Puertos locales específicos" pon: `3000`.
5. Selecciona **Permitir la conexión** > Siguiente.
6. Asegúrate de que Dominio, Privado y Público estén marcados.
7. Ponle de nombre "Servidor CIMA" y finaliza.

---

## 2. PONER EN MARCHA EL SERVIDOR

1. Copia la carpeta `server` al ordenador central.
2. Abre una terminal dentro de esa carpeta y ejecuta:
   ```bash
   npm install
   ```
3. Para que el servidor se inicie solo al encender el PC, instalaremos `pm2`:
   ```bash
   npm install -g pm2
   pm2 start index.js --name "cima-server"
   pm2 save
   ```

---

## 3. CONECTAR LA APP AL SERVIDOR

Debes actualizar la dirección en el código antes de generar la aplicación:

1. Busca el archivo: `constants/API.ts`
2. Cambia la IP por la que fijaste en el paso 1-B:
   ```typescript
   export const API_URL = 'http://192.168.1.100:3000/api';
   ```

---

## 4. GENERAR E INSTALAR LA APP (.APK)

1. **Instalar EAS CLI:** Ejecuta `npm install -g eas-cli`.
2. **Iniciar sesión:** `eas login`.
3. **Generar el archivo:**
   ```bash
   npx eas build -p android --profile preview
   ```
4. **Descargar e instalar:** Descarga el archivo `.apk` e instálalo en todas las tablets.

---

## 5. ACTUALIZACIONES AUTOMÁTICAS (OTA)

Para enviar cambios a las tablets sin tener que volver a instalar el APK:

1. **Configurar (Solo la primera vez):** Ejecuta `eas update:configure`.
2. **Enviar cambios:** Cada vez que modifiques el código, ejecuta:
   ```bash
   eas update --branch production --message "Descripción del cambio"
   ```

### ⚠️ ADVERTENCIA SOBRE ACTUALIZACIONES
No todos los cambios se pueden enviar por el aire. 
*   **Se PUEDEN actualizar:** Colores, textos, lógica de botones, nuevas pantallas, imágenes y arreglos de errores.
*   **NO se pueden actualizar (Requiere nuevo APK):** Cambios en el icono de la app, nombre de la app, permisos nuevos (ej: pedir permiso para usar Bluetooth si antes no se usaba) o añadir librerías que requieran configuración nativa.

*Regla de oro:* Si haces un cambio y al enviar el `eas update` la app no se actualiza o falla, genera un nuevo APK.

---

## 6. MANTENIMIENTO Y NOTAS

* **Base de Datos:** Toda la información se guarda en `server/database.sqlite`. Haz una copia periódica.
* **Conexión:** Las tablets deben estar en el mismo Wi-Fi que el servidor.
* **Actualizaciones:** Para recibir las actualizaciones OTA (paso 5), las tablets necesitan tener internet momentáneamente al abrir la App.
